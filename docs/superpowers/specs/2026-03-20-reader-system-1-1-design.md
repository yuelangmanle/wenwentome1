## 一、背景

`1.0` 已经完成云端打包、GitHub Release 与统一书库 MVP，但阅读系统仍有三类明显问题：

1. 本地 `TXT` 可读，但 `EPUB` 仍可能落到封面页或导航页，出现只显示 `Cover`、卡顿或无法进入正文
2. 书库界面仍偏原型态，缺少真正“像书架”的封面视觉、继续阅读和进度回流能力
3. 网文链路只完成了“搜索并加入书架”，尚未补齐详情、目录、正文、追更与进度保存闭环

用户已经明确希望本轮不是小修，而是一次“现代化、完整、可持续”的半重构升级。

## 二、已确认决策

### 2.1 整体实施强度

- 路线：`半重构`
- 保留现有 Kotlin + Compose + Room + DataStore + GitHub Sync 骨架
- 重做书架与阅读器 UI
- 重组本地书 / EPUB / 网文的统一阅读链路
- 不做彻底推翻式大重构

### 2.2 新视觉底座

- 首页底座：`书架首页`
- 视觉风格：`编辑精选感`
- 信息密度：`中密度平衡`
- 书籍封面形态：`长方形竖版封面，像真实书本，而不是普通卡片`

### 2.3 阅读器主交互

- 默认阅读模式：`仿真翻页`
- 同时支持：
  - `左右滑动`
  - `上下滚动`
- 三种模式共用一套正文数据与目录系统，只切换呈现方式，不拆成多套阅读器

### 2.4 网文目标深度

- 本轮目标不是“能搜到书”即可，而是完整网文系统
- 最低闭环必须包括：
  - 搜索
  - 书籍详情
  - 目录浏览
  - 正文阅读
  - 阅读进度保存
  - 刷新目录
  - 跳转最新章
  - 书架更新提示

### 2.5 封面策略

每本书都支持以下四种封面状态：

1. `自动获取`
2. `手动更新`
3. `手动导入照片`
4. `恢复自动封面`

自动获取优先级：

1. EPUB 内嵌封面
2. 本地书元数据可推导封面
3. 网文详情页封面
4. 默认占位封面

## 三、目标与非目标

### 3.1 本轮目标

本轮升级必须同时达成以下结果：

1. 修掉 EPUB 首次打开落到 `Cover` 或非正文资源的问题
2. 把书架首页升级为现代化、封面驱动、可回流阅读进度的主界面
3. 把阅读器升级为支持三种翻页模式、目录、阅读进度和基础阅读设置的正式阅读器
4. 把网文从“加入书架”补齐为“可追更、可继续阅读”的完整链路
5. 让本地书、EPUB、网文共用统一书架、统一目录模型、统一阅读进度与统一封面策略

### 3.2 非目标

本轮不追求以下内容：

- `PDF` 阅读支持
- 在线封面云服务
- 复杂推荐算法
- 网文全量离线缓存所有章节
- 推送通知或后台自动追更
- 多端阅读进度实时合并冲突策略升级

## 四、产品结构重组

### 4.1 顶层导航保持四块能力，但重新分工

顶层仍以现有 `书架 / 发现 / 我的` 为主，不新增新的顶层 tab。

重组后的职责如下：

#### 书架

书架不再只是书目列表，而是整套阅读系统的中枢页面，负责：

- 继续阅读
- 全部书籍浏览
- 本地书与网文统一展示
- 阅读进度回流
- 最近更新提示
- 分类、排序、筛选
- 导入入口

#### 阅读

阅读仍然是书架进入的深层页面，但能力显著增强，负责：

- 仿真翻页 / 左右滑动 / 上下滚动三模式
- 目录
- 进度保存
- 章节跳转
- 阅读主题、字号、行距等基础阅读设置
- 书签与后续增强入口

#### 发现

发现不再只是搜书入口，而是完整网文系统的前台，负责：

- 搜索
- 书籍详情
- 目录预览
- 开始阅读
- 加入书架
- 刷新目录
- 跳转最新章
- 书源管理

#### 我的

继续承载：

- GitHub 同步
- 更新日志
- 项目与作者信息
- 未来阅读偏好等低频配置

### 4.2 首页骨架

书架首页采用中密度结构：

1. 顶部标题区
2. `继续阅读` 主卡片
3. `全部书籍` 封面书架栅格
4. 分类 / 排序入口
5. 底部导航

视觉要求：

- 书籍必须是长方形竖版封面
- 封面具备“像书”的视觉直觉，包括书脊或厚度暗示
- 整体保留编辑精选感，而不是复古木纹书房风
- 既现代又能让人一眼认出“这是我的书架”

## 五、统一阅读模型

### 5.1 顶层对象保持 `BookRecord`

延续 `1.0` 的核心原则：

- 顶层第一公民仍然是书架中的一本书
- 来源不是顶层对象
- 书源、EPUB、TXT 都先落成统一书目，再进入阅读系统

### 5.2 新增统一目录模型

本轮需要把三类内容接到同一目录抽象下：

- `TXT`：基于章节识别规则切目录
- `EPUB`：优先使用 `TOC/nav`，回退到过滤后的 `spine`
- 网文：使用书源章节列表

规划中的统一目录条目至少需要具备：

- `chapterRef`
- `title`
- `orderIndex`
- `sourceType`
- `locatorHint`
- `isLatest`

这样阅读器、目录页、进度恢复与网文更新检测才能共用同一套结构。

### 5.3 统一阅读进度

阅读状态至少统一保存以下字段：

- `bookId`
- `chapterRef`
- `locator`
- `progressPercent`
- `updatedAt`

可复用现有 `ReadingState` 主体，但本轮要把它真正贯穿到：

- 继续阅读卡片
- 章节定位
- 三种翻页模式
- 网文继续追读
- 书架排序

## 六、EPUB 设计修复

### 6.1 当前问题

当前 `EPUB` 阅读直接按 `spine` 资源读取，容易把以下非正文资源当首章：

- 封面页
- 导航页
- 版权页
- 空 HTML 资源

这会导致：

- 首次进入只看到 `Cover`
- 阅读器看似卡住，实则进入了错误资源
- 没有可用目录

### 6.2 修复原则

首次阅读 EPUB 时，默认进入“第一章正文”，而不是“第一个资源”。

### 6.3 处理规则

正文入口规则如下：

1. 优先读取 EPUB 的 `TOC/nav`
2. 找到第一个可读章节并作为默认起点
3. 若 `TOC/nav` 缺失，再回退到 `spine`
4. 回退时跳过：
   - 封面页
   - 导航页
   - 版权页
   - 空内容页
5. 打开时若存在历史进度，则优先恢复历史进度

### 6.4 封面解析

EPUB 自动封面优先使用内嵌封面资源，并与正文入口完全解耦：

- 封面只服务于书架展示
- 封面资源不再默认作为正文入口

## 七、阅读器设计

### 7.1 阅读模式

阅读器支持三种模式：

#### 模式 A：仿真翻页

- 本轮默认模式
- 用于营造“像在读一本书”的沉浸感
- 适合作为 EPUB 和长篇小说的主展示模式

#### 模式 B：左右滑动

- 较稳妥的分页模式
- 兼容本地书与网文章节阅读
- 适合不喜欢仿真动画的用户

#### 模式 C：上下滚动

- 最适合长时间阅读网文
- 适合连载追更与快速浏览

### 7.2 模式切换原则

- 模式切换入口放在阅读菜单中
- 切换后即时生效
- 支持记住全局默认偏好
- 允许未来扩展为“按书记忆偏好”

### 7.3 阅读菜单

阅读器菜单至少包含：

- 目录
- 翻页模式切换
- 字号
- 行距
- 亮度 / 主题
- 阅读进度显示

### 7.4 目录页

目录页必须支持：

- 章节列表
- 当前章节高亮
- 点击跳转
- 网文最新章定位
- 返回阅读器后保持当前位置

## 八、封面系统设计

### 8.1 封面来源

每本书封面有三种来源入口和一种回退策略：

#### 自动获取

- EPUB：取内嵌封面
- 本地 TXT：优先元数据或默认封面
- 网文：取详情页封面

#### 手动更新

- 重新抓取网络封面
- 或重新解析书籍内嵌封面

#### 手动导入照片

- 从相册或本地文件选择图片

#### 恢复自动封面

- 删除当前手动覆盖图
- 回退到自动封面逻辑

### 8.2 入口位置

封面操作入口放两处：

- 书架长按菜单
- 书籍详情页

这样可以保持首页整洁，同时让操作可达。

## 九、网文完整链路

### 9.1 必备闭环

网文从 `1.0` 的“加书架”补齐为完整流程：

1. 搜索书籍
2. 进入书籍详情
3. 查看简介、作者、封面与最新章
4. 查看目录
5. 加入书架
6. 开始阅读
7. 保存阅读进度
8. 从书架继续阅读
9. 刷新目录
10. 跳转到最新章
11. 在书架显示“有更新”

### 9.2 本轮范围边界

本轮“完整网文系统”的定义是完整阅读闭环，不等于把所有高级网文能力一次做满。

本轮建议范围内：

- 搜索
- 详情
- 目录
- 正文
- 追更状态
- 继续阅读
- 阅读进度保存

本轮范围外：

- 后台自动轮询更新
- 推送提醒
- 全章节离线下载器

## 十、实施路线

### 10.1 路线原则

虽然用户选择的是“1.1 大改版”，但实施应采用“阅读内核先稳住，再整体换新壳”的顺序。

原因：

1. EPUB `Cover` 问题是当前实际阻塞
2. 阅读器目录与进度是书架首页成立的前提
3. 若先大改外壳，再回头补阅读内核，会出现重复返工

### 10.2 分阶段拆分

#### 阶段 A：阅读内核与数据模型

- EPUB 正文入口修复
- 统一目录模型
- 阅读进度贯通
- 三种翻页模式骨架
- 网文章节正文链路打通

#### 阶段 B：书架与详情页重做

- 新书架首页
- 长方形书封视觉
- 继续阅读卡片
- 进度与更新状态回流
- 封面管理入口

#### 阶段 C：发现页与追更体验

- 网文详情页
- 目录预览
- 最新章跳转
- 更新状态展示

#### 阶段 D：打磨与验证

- 真机滚动与翻页手感
- EPUB 兼容性回归
- 网文不同书源适配
- 视觉统一与动效收尾

## 十一、涉及模块

预计会影响以下模块：

- `data/localbooks`
  - EPUB / TXT 解析
- `feature/reader`
  - 阅读器、目录、翻页模式、阅读设置
- `feature/library`
  - 新书架首页、封面展示、进度回流
- `feature/discover`
  - 网文详情、目录、追更链路
- `bridge/source`
  - 目录 / 正文章节能力补齐
- `core:model`
  - 统一目录与封面相关模型
- `core:database`
  - 阅读状态与封面覆盖信息的持久化

## 十二、验证要求

本轮实现完成后，至少要验证以下场景：

1. EPUB 首次打开进入正文，不再停在 `Cover`
2. 三种翻页模式都能进入同一本书并正确保存进度
3. 目录跳转后，阅读器可以回到对应章节
4. 网文从搜索到书架再到阅读的闭环可用
5. 书架首页可以正确显示封面、进度和最近更新
6. 手动导入封面、手动更新封面、恢复自动封面可用

## 十三、最终结论

`1.1` 的本质不是“换一层皮”，而是把当前的本地书、EPUB 与网文阅读重新收束为一套真正统一的阅读系统：

- 统一书架
- 统一目录
- 统一进度
- 统一封面
- 统一阅读入口

只要这个设计成立，后续 `1.2+` 才有资格继续扩展阅读设置、缓存、批注和更多网文能力，而不会继续在割裂的底层上堆功能。
