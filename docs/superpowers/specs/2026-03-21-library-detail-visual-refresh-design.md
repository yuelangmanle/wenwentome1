## 一、背景

当前 `1.1` 已经把统一书库、阅读模式、网文目录与精确续读主链打通，功能层面已经具备“能读、能续、能回流”的基础闭环，但书架与书籍详情页仍存在三类明显短板：

1. 书架页虽然可用，但视觉上仍偏“组件堆叠”，没有形成明确的主次层次和真正像书架的氛围
2. 书籍卡片虽然已经切到长方形封面，但仍更像普通内容卡，缺少“立起来的书”和实体藏书感
3. 书籍详情页功能齐全但重点分散，封面、简介、进度、目录和封面管理没有形成稳定的信息层级

用户已经明确确认，本轮要做的不是简单换皮，而是在不破坏当前阅读主链的前提下，把书架页与详情页升级成更完整、更现代、也更像书的界面。

## 二、已确认决策

### 2.1 主视觉路线

- 采用：`轻拟物精装书城版`
- 目标不是复古木纹，而是“现代排版里的书本感”
- 视觉气质要求：
  - 有书本立体感
  - 有适度层次和光影
  - 不走过重的复古装饰
  - 仍保持阅读产品应有的清爽与可读性

### 2.2 书架页氛围

- 顶部氛围：`混合`
- 上半区采用“书桌陈列感”
- 下半区采用“藏书墙感”

这意味着书架页首屏既要有一个明确的主视觉焦点，也要让书籍网格本身像一个真正的书架，而不是普通列表。

### 2.3 详情页首屏优先级

- 首屏优先：`书籍展示优先`
- 首屏重点顺序：
  1. 大封面
  2. 书名 / 作者 / 简介
  3. 主操作按钮 `继续阅读 / 开始阅读`

说明：

- 继续阅读仍然是详情页最重要的操作
- 但在视觉上不压过作品展示区
- 详情页需要更像“一本书的介绍页”，而不是“功能按钮集合页”

### 2.4 封面管理能力

封面相关入口必须继续保留，并统一整理为一个稳定操作区。

为避免后续计划误解，本轮统一使用以下固定术语：

1. `刷新自动封面`
   - 含义：用户主动触发一次自动封面重新获取
   - 它对应当前已有的自动封面刷新能力
   - 本轮不新增一个与之平行的“手动更新封面协议”
2. `导入照片`
   - 含义：用户从本地选择图片作为手动封面
3. `恢复自动封面`
   - 含义：清除当前手动封面，回退到自动封面来源
   - 仅在存在手动封面时显示

结论：

- 用户此前口头表达里的“手动更新封面”，在本轮实现中统一收敛为 `刷新自动封面`
- 封面管理入口继续放在书籍详情页，而不是分散回书架卡片或书架长按菜单

## 三、目标与非目标

### 3.1 本轮目标

本轮改版必须同时达到以下目标：

1. 让书架页一眼看上去像“真正的书架”，而不是普通内容栅格
2. 让每本书的封面卡片具备更明确的“书本感”，包括长方形比例、书脊暗示、层次与状态角标
3. 让顶部继续阅读区域成为书架页的视觉焦点，并能更清楚表达最近阅读与当前进度
4. 让详情页首屏形成“封面展示优先”的稳定层级
5. 让阅读进度、目录入口、刷新目录、跳转最新章、封面管理等操作形成更清楚的分区
6. 不破坏当前 `TXT / EPUB / WEB` 统一阅读状态、继续阅读与精确续读链路

### 3.2 非目标

本轮不做以下扩展：

- 新增新的顶层 tab
- 改造阅读器正文页的翻页逻辑或阅读协议
- 改写 `ReaderLocator`、章节恢复或阅读进度存储协议
- 新做一套独立的“网文首页”
- 引入在线封面服务或远程图片托管能力
- 推进 `PDF` 阅读支持

## 四、体验与信息架构设计

### 4.1 书架页结构

书架页采用“两段式”结构：

1. 顶部 `书桌陈列区`
2. 下方 `藏书墙区`

#### 顶部书桌陈列区

这一块承载当前主视觉与主操作，包含：

- 页面标题与辅助状态
- 一张强化版 `继续阅读卡`
- 最近阅读信息
- 当前阅读进度
- 必要的更新提示

设计要求：

- 视觉上像一本当前正被阅读的主书放在桌面上
- 左侧强调主书封面，右侧承载书名、作者、最近阅读、进度与主按钮
- 这块必须明显区别于下面的普通书籍网格，承担首屏焦点作用

#### 下方藏书墙区

这一块承载全部书籍，使用封面驱动栅格布局：

- 本地书与网文统一展示
- 每本书优先展示封面，不再做大面积文字卡
- 书名、作者、进度作为封面下方的辅助层

设计要求：

- 网格中每本书都应该像立起来的书，而不是内容缩略图
- 网格需要兼顾密度和呼吸感，避免“卡片挤满屏”
- 长按操作入口保留，但只作为轻量二级入口
- 长按菜单本轮不承载封面管理能力
- 长按菜单只允许保留不破坏职责边界的快捷动作，例如：
  - 打开详情页
  - 网文书籍的 `刷新目录`

### 4.2 书籍卡片设计

单本书卡片由三层构成：

1. 封面层
2. 状态层
3. 元信息层

#### 封面层

- 保持接近真实书本的长方形纵向比例
- 左侧增加轻微书脊暗示
- 通过高光、边缘阴影和投影强化“立体书”观感
- 有真实封面时优先展示封面图
- 无封面时使用本地样式化占位封面，但仍保留书本质感

#### 状态层

- 更新状态以角标形式置于封面右上角
- 来源类型使用轻量标签，例如 `本地`、`网文`
- 已读进度显示在封面下方信息层，不压在封面主体上

#### 元信息层

- 书名最多两行
- 作者一行
- 进度单独一行
- 文字信息不再喧宾夺主，主要服务于浏览确认

### 4.3 继续阅读卡设计

继续阅读卡作为书架页主视觉，需要比普通书籍卡更像“桌面陈列”：

- 左侧放较大的当前书封面
- 右侧放标题、作者、最近阅读、阅读进度、主按钮
- 可以附带简短的状态信息，例如 `上次读到第 12 章` 或 `有 3 章更新`

交互要求：

- 点击卡片整体或主按钮，均进入继续阅读
- 保持与当前 `onContinueReadingClick` 一致，不增加额外路由复杂度

### 4.4 详情页结构

详情页采用四段结构：

1. 首屏作品展示区
2. 阅读状态区
3. 封面管理区
4. 目录区

#### 首屏作品展示区

包含：

- 大封面
- 书名
- 作者
- 来源或最近阅读标签
- 简介摘要
- 主操作按钮 `开始阅读 / 继续阅读`

设计要求：

- 大封面是首屏最强视觉元素
- 简介需要足够可读，但控制在短摘要级别，避免首屏过长
- 按钮仍然要容易点击，但视觉顺位低于封面主体

#### 阅读状态区

集中承载：

- 阅读进度
- 最近阅读时间
- 当前章节信息
- `查看目录`
- `刷新目录`
- `跳转最新章`

设计要求：

- 状态与操作放在同一区块，避免散落在多个卡片里
- 网文书籍按现有能力显示 `刷新目录 / 跳转最新章`
- 本地书不展示不相关操作

#### 封面管理区

封面管理统一整理为一个单独操作区，避免按钮零散分布：

- 刷新自动封面
- 导入照片
- 恢复自动封面（条件显示）

设计要求：

- 该区块在信息层级上早于目录区
- 这样既不会抢首屏主视觉，也不会埋得太深

#### 目录区

- 默认折叠，由按钮展开
- 展开后显示章节列表
- 当前章与最新章需要容易分辨
- 除非现有 `UiState` 已能直接推导，否则本轮不新增章节级“未读追踪”状态

设计要求：

- 目录视觉上应当像一块清晰的章节面板
- 不在目录区塞入额外复杂功能，保持聚焦

## 五、系统边界与数据流

### 5.1 保持现有阅读状态主链

本轮只升级 UI 呈现，不重写阅读状态协议。

书架页、继续阅读卡、详情页三处都继续读取当前统一的阅读状态来源，包括：

- `progressPercent`
- `progressLabel`
- `lastReadLabel`
- 当前章节信息
- `WEB` 章内段落级恢复结果

结论：

- 本轮不新增一套独立状态模型
- 优先基于现有 `UiState` 组合结果做展示重排
- 目录项只消费当前已有的章节列表、当前章节与最新章节信息
- 本轮不为目录额外引入“逐章未读 / 已读”持久化逻辑

### 5.2 本地书与网文的显示差异

本地书与网文继续共用同一套书架与详情页结构，但展示层做差异化：

- 本地书更强调：
  - 封面
  - 作者
  - 阅读进度
- 网文更强调：
  - 更新状态
  - 最新章节提示
  - 刷新目录与跳转最新章能力

说明：

- 不再为网文另做完全独立的书架样式
- 差异只存在于标签、操作与状态文案

### 5.3 保持当前交互路由

本轮继续沿用当前入口分工：

- 书架卡片点击进入书籍详情
- 继续阅读卡直接进入阅读
- 详情页按钮负责继续阅读、目录展开与封面管理

结论：

- 本轮不重做导航结构
- 重点是重排同一路由下的视觉和信息层级

## 七、错误处理与降级策略

### 7.1 封面缺失

当没有可用封面时：

- 仍显示具有书本感的占位封面
- 不出现空白图片区
- 占位封面需要能够承载书名与基础状态

### 7.2 信息缺失

当作者、简介、最近阅读等字段缺失时：

- 使用当前已有的兜底文案
- 保持版式完整，不因字段缺失导致布局断裂

### 7.3 网文操作缺失

当书籍不是网文来源时：

- 不展示 `刷新目录`
- 不展示 `跳转最新章`
- 目录区仍可保留阅读章节展示，但不附带网文更新动作

### 7.4 小屏兼容

在窄屏下需要保持以下原则：

- 继续阅读卡允许垂直堆叠
- 详情页首屏允许封面与文本改为纵向排列
- 书架页网格列数随屏宽自适应

## 八、测试与验证策略

### 8.1 验证目标

本轮验证重点不是阅读协议，而是确保 UI 改版不破坏当前稳定链路：

1. 书架页和详情页仍可正常进入阅读
2. 继续阅读入口仍然可用
3. 目录展开、章节跳转、刷新目录和跳转最新章入口仍然可用
4. 关键测试 tag 不因纯视觉改版而丢失
5. 已稳定的 `WEB` 精确续读与书架回流测试不回退

### 8.2 测试方式

因为本机仍缺少 Java 运行时，本轮继续以 GitHub Actions 为主做验证。

预期做法：

- 保留并必要时更新现有 Compose UI 测试
- 保留 `ReaderScreenTest`、`AppReaderFlowTest` 等主链测试
- 对因为视觉层级调整而引发的脆弱断言，优先做测试收敛，不绕开真实交互

### 8.3 完成标准

本轮可以进入实现的前提是：

- 书架页已切到“顶部陈列区 + 下方藏书墙”结构，并保留 `continue-reading-card`
- 书籍卡片维持稳定纵向书本比例，并继续保留 `book-cover-card-<id>` 测试入口
- 详情页已切到“作品展示区 / 阅读状态区 / 封面管理区 / 目录区”的四段结构
- `detail-read-button` 与 `detail-progress-label` 继续保留，避免破坏现有 UI 测试定位
- 封面管理动作收敛为 `刷新自动封面 / 导入照片 / 恢复自动封面`
- 继续阅读、目录展开、网文刷新目录与跳转最新章主链保持可用
- 现有阅读主链与云端 CI 保持稳定

## 九、实施建议

建议的实施顺序如下：

1. 先改 `BookCoverCard` 与 `ContinueReadingCard`，建立新的书本视觉语言
2. 再改 `LibraryScreen`，完成书桌陈列区与藏书墙布局
3. 然后改 `BookDetailScreen`，重排首屏、状态区、封面管理区与目录区
4. 最后根据 UI 结构变化，修正或补充测试

这样做的原因是：

- 视觉语言先稳定，页面重排会更顺
- 书架和详情页共享的“书本感”组件可以先沉淀
- 避免先改大页面，再在内部多次返工
